# Weekly GitHub Skills Learning Plan

**Time Commitment**: 1 hour/day (weekdays) + 4 hours (weekends)
**Total Weekly Time**: 9 hours
**Course**: Introduction to GitHub

---

## Week 1: GitHub Fundamentals

### Monday (1 hour)
**Topic**: Getting Started & Understanding GitHub Basics

- [ ] Read the course introduction and overview (15 min)
- [ ] Watch: "What is GitHub?" video (10 min)
- [ ] Read about repositories, branches, and version control concepts (20 min)
- [ ] Review GitHub glossary terms: repository, branch, commit, pull request (15 min)

**Resources**:
- README.md (main course file)
- Step 1 content (.github/steps/1-create-a-branch.md)
- GitHub Docs: [About repositories](https://docs.github.com/en/repositories/creating-and-managing-repositories/about-repositories)

---

### Tuesday (1 hour)
**Topic**: Step 1 - Create a Branch

- [ ] Review what branches are and why they're important (15 min)
- [ ] Complete Activity: Create your first branch named `my-first-branch` (20 min)
- [ ] Read about profile READMEs and their purpose (10 min)
- [ ] Experiment: Try creating another test branch for practice (15 min)

**Key Concepts**:
- Parallel development with branches
- Keeping work separate from the main branch
- Branch naming conventions

---

### Wednesday (1 hour)
**Topic**: Step 2 - Commit a File

- [ ] Read about commits and what they represent (10 min)
- [ ] Learn about Markdown (.md files) basics (15 min)
- [ ] Complete Activity: Create PROFILE.md file on your branch (20 min)
- [ ] Practice: Make your first commit with message "Add PROFILE.md" (15 min)

**Key Concepts**:
- What is a commit?
- Writing good commit messages
- Creating and editing files in GitHub

**Additional Resource**: [Basic writing and formatting syntax](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax)

---

### Thursday (1 hour)
**Topic**: Step 3 - Open a Pull Request

- [ ] Read about pull requests and collaboration (15 min)
- [ ] Complete Activity: Create your first pull request (20 min)
- [ ] Write a descriptive PR title: "Add my first file" (5 min)
- [ ] Practice writing a good PR description summarizing your changes (10 min)
- [ ] Review the side-by-side comparison of changes (10 min)

**Key Concepts**:
- Pull requests as collaboration tools
- Comparing changes between branches
- Communicating changes to others

---

### Friday (1 hour)
**Topic**: Step 4 - Merge Your Pull Request

- [ ] Read about merging and what it means (15 min)
- [ ] Understand GitHub Actions notifications (10 min)
- [ ] Complete Activity: Merge your pull request (15 min)
- [ ] Clean up: Delete the merged branch (5 min)
- [ ] Review what happens after a merge (15 min)

**Key Concepts**:
- Incorporating changes into main branch
- Branch cleanup after merging
- Understanding merge conflicts (theory)

---

### Weekend (4 hours total)

#### Saturday (2 hours)
**Topic**: Practice & Profile README Project

- [ ] Review all concepts learned during the week (30 min)
- [ ] Read: [Managing your profile README](https://docs.github.com/account-and-profile/setting-up-and-managing-your-github-profile/customizing-your-profile/managing-your-profile-readme) (20 min)
- [ ] Project: Create your personal GitHub profile repository (30 min)
- [ ] Project: Design and write your profile README.md (40 min)

**Goal**: Create a public repository matching your GitHub username with a personalized README

---

#### Sunday (2 hours)
**Topic**: Additional Practice & Exploration

- [ ] Complete the "Communicating using Markdown" course (if interested) (60 min)
- [ ] Explore GitHub Explore to find interesting projects (20 min)
- [ ] Read about contributing to open source (20 min)
- [ ] Try: Fork a simple repository and make a practice contribution (20 min)

**Resources**:
- [GitHub Skills](https://github.com/skills) - More courses
- [GitHub Explore](https://github.com/explore)
- [Student Developer Pack](https://education.github.com/pack) (if applicable)

## Tips for Success

1. **Take Notes**: Keep a learning journal of new terms and concepts
2. **Practice Regularly**: Consistency is key - try to study at the same time each day
3. **Don't Rush**: Take time to understand each concept before moving on
4. **Ask Questions**: Use the [GitHub Skills discussion board](https://github.com/orgs/skills/discussions/categories/introduction-to-github) if you get stuck
5. **Experiment**: Don't be afraid to try things - you can always create test repositories
6. **Review**: Spend the first 5-10 minutes of each session reviewing what you learned previously
